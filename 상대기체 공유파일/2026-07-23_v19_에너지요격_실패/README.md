# v19 — 에너지 축적 요격 (실패, 전제 반증됨)

## 문제 인식 (6/6 리허설 시드 비교로 규명)
승패가 "dead-six 추격이 어느 거리에서 고착되나"로 갈렸다.
- seed2(격추): 750~850m 고착 = 사거리 안 → 132초 누적 데미지로 격추
- seed0(무득점): 같은 추격이 1400~1900m 고착, 상대는 상승 도주(3.2→15.8km),
  우리도 따라 올라 130초 낭비. 동일 기체 풀스로틀 직선 추격 = dV 0 = 거리 동결.

## 가설과 구현
"상대가 상승으로 에너지를 고도에 버리는 동안 우리는 따라 오르지 않고 수평 가속하면
속도 우위(dV)가 생긴다" → 교착 감지 시 VP 수직을 내고도+60m로 캡, dV+45 확보 후 요격.

## 결과 — 전제가 이 시뮬레이터에서 성립하지 않음
- **v18 seed0에서 둘 다 풀스로틀로 95~107m/s 상승 중에도 dV≈0이었다.**
  = 이 F-16 모델은 지속 상승이 속도를 거의 깎지 않는다. **"상승 = 에너지 낭비" 자체가 거짓.**
- v19 seed0 실측: eco 발동 중 dV −19~−27 (우리가 더 느림!), 거리 1390→4576m 확대,
  상대가 1438m 위로 분리. reward 755→42. 명백한 악화.
- zCap=+60m/틱은 "수평"이 아니라 "완만 상승"이라 가속 효과도 없었다 (설계 오류 이중).

## 배치 +64%(dealt 1.19→1.95)는 왜 무시하나
권정환 DLL이 에피소드 간 내부 상태를 유지해, seed0의 판도가 바뀌면 seed1~4 전부가
연쇄 오염된다(단독 seed2 ≠ 배치 seed2로 실증). 오염 없는 단독 seed0 비교에선 v19가
명백히 나쁘다. **메커니즘이 반증된 이상 점수 우연으로 채택하지 않는다** (사용자 원칙).

## 남긴 교훈
1. **이 시뮬에선 에너지 트릭(상승 유도 후 가속)이 무효.** 교착 해제는 다른 길로:
   후보 = 교착을 "만들지 않기" — 상대 six 진입 시 에너지를 덜 잃는 lag entry
   (회전각↓속도↑, 사용자 사양 2번), 고착 전 폐쇄율을 더 깊이 끌고 가기.
2. **시드 배치도 순서 오염이 있다.** 깨끗한 개별 비교는 시드마다 새 프로세스로.
3. 검증 표준(rehearsal 6/6)이 이 실패를 하루 안에 잡아냈다 — 표준 유효.
