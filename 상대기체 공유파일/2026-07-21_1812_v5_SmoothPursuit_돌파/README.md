# v5 SmoothPursuit — 돌파 버전 (2026-07-21)

## 정체 (중요)
**"단순화 = 강력함"을 증명한 첫 승리 버전.** 팀원의 복잡한 트리(v0~v2)와 우리
v3/v4(노드 추가)가 전부 승리 0건이었는데, **트리를 극단적으로 단순화**하니 처음으로
뒤잡기·조준·사격이 모두 성공했다.

## 핵심 설계
- 트리 = **인지 + "고도안전 / 추격" 두 상태만** (노드 전환 최소화)
- `Task_SmoothPursuit`: VP=적 현재위치(pure pursuit) + **풀스로틀(에너지 유지)**
- 감속·방어·BFM 우선순위 전부 제거

## 왜 됐나 (진단 근거)
복잡한 트리는 매 틱 노드 전환(GetTail↔Lead↔LagEntry↔MergeReversal)으로 VP가 튀어
기동이 불안정(속도 208~358, 선회율 -2~19 요동) → 급기동으로 에너지 낭비 → 저속
표적조차 못 잡음. 단일 매끄러운 추격은 에너지를 안 잃어 뒤에 안정적으로 붙는다.

## 한계 (다음 개선)
1. **완전 격추는 못 함** (24~30% 데미지). 사격 지속 효율 개선 필요.
2. **방어 없음** — 상대가 공격적이지 않아 무손상이었을 뿐, 회피·반격하는 실전
   상대엔 취약할 수 있음. 단, 방어 추가는 노드 전환을 늘려 불안정 위험 → 신중히
   (히스테리시스로 추격↔방어 전환 억제).
3. 대회 실전 상대(회피 기동)로 검증 필요.
